Both the 'upscale' and 'upscale higher' files use a state machine algorithm to do something akin to DLSS but with perfect pixels;
it creates a wavemap data file that can reconstruct the original resolution.

This can be compressed further.

The logic of this could be baked into a game engine to render a very high res (think 8k) renderer into a lower resolution output.

Or it could use the info with a fast cpu but low resolution screen to perfectly replicate an 8k display's pixel dense information.

=

The upscale program needs a JPG image, called 'green-field-with-yellow-dandelions' and it will cut it in half, and create a wave map.

The upscale higher program reverts this back to the original high resolution.
